# 第03回 物理モデルとしてのHamiltonian

## はじめに

前回は：

* Pauli演算子とは何か
* テンソル積の意味
* Pauli文字列
* Hamiltonianの一般形
* Isingモデルの基本構造
* VQE / QAOAとの関係
* 期待値計算の分解方法

を整理しました。

ここまでで：

Hamiltonianを

**量子回路として扱う準備**

が整いました。

今回は：

実際に物理で使われるHamiltonianが

どのような構造を持つのか

を整理します。

つまり：

**現実世界の問題がどのように量子回路になるのか**

を理解します。

---

# なぜ物理モデルを学ぶのか？

Hamiltonian simulationの目的は：

現実の量子系を再現すること

です。

対象例：

電子
スピン
原子
分子
材料

これらはすべて：

Hamiltonianで記述されます。

---

# 最も基本的な物理モデル：スピン系

量子シミュレーションの出発点は：

スピンモデル

です。

理由：

数学的に扱いやすい
量子回路に変換しやすい
応用範囲が広い

からです。

---

# スピンとは何か？

スピンとは：

粒子が持つ内部自由度

です。

量子ビットでは：

```text
|0⟩ = スピン上向き
|1⟩ = スピン下向き
```

として表現できます。

つまり：

量子ビット = スピン系モデル

です。

---

# 最も単純なモデル：単一スピン

最小のHamiltonian：

```text
H = Z
```

意味：

z方向の磁場中のスピン

です。

これは：

磁場とスピンの相互作用

を表します。

---

# 外部磁場との相互作用

一般形：

```text
H = hZ
```

ここで：

h = 磁場強度

です。

磁場が強いほど：

エネルギー差が大きくなります。

---

# 複数スピンの相互作用

複数粒子では：

粒子同士が影響し合います。

例：

```text
H = Z₀Z₁
```

意味：

2つのスピンの相互作用

です。

これが：

多体系モデルの基本単位

になります。

---

# Isingモデルとは何か？

最も重要なスピンモデル：

Isingモデル

です。

一般形：

```text
H = Σ Jᵢⱼ ZᵢZⱼ + Σ hᵢ Zᵢ
```

ここで：

Jᵢⱼ = 相互作用強度
hᵢ = 外部磁場

です。

---

# Isingモデルの直感的理解

このモデルは：

スピンが揃いたがるか

逆向きになりたがるか

を表します。

例えば：

```text
H = Z₀Z₁
```

なら：

|00⟩ と |11⟩ が安定
|01⟩ と |10⟩ が不安定

になります。

---

# なぜIsingモデルが重要なのか？

理由：

多くの問題が

Ising Hamiltonian

に変換できるからです。

例：

組合せ最適化
MaxCut問題
巡回セールスマン問題
スケジューリング問題

これらは：

量子最適化に直結します。

---

# QAOAとの関係

QAOAでは：

問題そのものを

Hamiltonianとして書きます。

例：

```text
H = Z₀Z₁ + Z₁Z₂
```

これは：

最適化対象のコスト関数

です。

つまり：

最適化問題 = 物理モデル

になります。

---

# Heisenbergモデルとは何か？

次に重要なモデル：

Heisenbergモデル

です。

一般形：

```text
H = X₀X₁ + Y₀Y₁ + Z₀Z₁
```

これは：

3方向すべての相互作用

を表します。

---

# Isingモデルとの違い

Isingモデル：

```text
Z方向のみ
```

Heisenbergモデル：

```text
X・Y・Zすべて
```

つまり：

より現実に近いモデル

です。

---

# Heisenbergモデルの用途

応用例：

磁性体
量子相転移
スピン鎖
量子情報伝播

などがあります。

---

# XYモデルとは何か？

もう一つ重要なモデル：

XYモデル

です。

例：

```text
H = X₀X₁ + Y₀Y₁
```

これは：

スピン交換相互作用

を表します。

---

# スピン交換とは何か？

意味：

スピンが入れ替わる

相互作用です。

例：

```text
|01⟩ → |10⟩
```

これは：

量子情報伝送

に関係します。

---

# スピン鎖モデルとは何か？

複数スピンが

一直線に並んだモデル：

スピン鎖

です。

例：

```text
H = Z₀Z₁ + Z₁Z₂ + Z₂Z₃
```

これは：

多体系シミュレーションの基本モデル

です。

---

# なぜスピン鎖が重要なのか？

理由：

量子多体系の最小モデル

だからです。

応用例：

量子相転移
量子通信
材料物性解析

があります。

---

# 分子Hamiltonianへの接続

スピンモデルは：

分子モデルの近似

として使えます。

分子では：

電子同士が相互作用します。

これも：

Hamiltonianで記述できます。

例：

```text
H = Σ cᵢ Pᵢ
```

ここで：

Pᵢ = Pauli文字列

です。

---

# なぜ分子Hamiltonianが難しいのか？

理由：

電子数が多い
相互作用が複雑
自由度が大きい

からです。

そのため：

量子コンピュータが期待されています。

---

# 材料モデルへの接続

材料科学では：

多数粒子系

を扱います。

例：

格子モデル

```text
H = Σ ZᵢZⱼ
```

これにより：

磁性
伝導性
超伝導

などを解析できます。

---

# 量子アルゴリズムとの統一的理解

ここまでのモデル：

Ising
Heisenberg
XY

はすべて：

Pauli文字列の線形結合

です。

つまり：

量子回路として実装可能

です。

---

```text

```

を計算することです。

つまり：

物理モデルの時間発展

を再現することです。
