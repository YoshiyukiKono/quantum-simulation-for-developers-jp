# 第04回 時間発展演算子とは何か

## はじめに

前回は：

* スピンモデル
* Isingモデル
* Heisenbergモデル
* XYモデル
* スピン鎖モデル
* 分子Hamiltonian
* 材料モデル

を整理しました。

ここまでで：

現実の物理問題が

Hamiltonianとして記述される構造

が見えてきました。

今回は：

量子シミュレーションの核心である

```text
U(t) = exp(-iHt)
```

を理解します。

つまり：

**なぜ量子状態の変化が指数関数で表されるのか**

を解説します。

---

# 時間発展とは何か？

自然界の量子状態は：

時間とともに変化します。

例えば：

電子の運動
スピンの回転
分子振動
波動関数の広がり

これらはすべて：

時間発展

です。

---

# 量子状態の時間変化

量子状態：

```text
|ψ(t)⟩
```

は：

時間とともに変化します。

そして：

その変化を決めるのが

Hamiltonian

です。

---

# シュレディンガー方程式

量子状態の時間変化は：

次の式で表されます：

```text
i d|ψ(t)⟩/dt = H |ψ(t)⟩
```

これが：

シュレディンガー方程式

です。

---

# この式が意味すること

この式は：

状態の変化の速さ

が

Hamiltonianによって決まる

ことを意味します。

つまり：

Hamiltonian = 状態変化のルール

です。

---

# 微分方程式として見る

シュレディンガー方程式は：

次の形をしています：

```text
d|ψ⟩/dt = -iH |ψ⟩
```

これは：

一次線形微分方程式

です。

そして：

このタイプの解は

指数関数になります。

---

# 解が指数関数になる理由

一般に：

```text
dx/dt = Ax
```

という方程式の解は：

```text
x(t) = exp(At)x(0)
```

になります。

同様に：

```text
|ψ(t)⟩ = exp(-iHt)|ψ(0)⟩
```

となります。

---

# 時間発展演算子の定義

そこで：

次の演算子を定義します：

```text
U(t) = exp(-iHt)
```

これを：

時間発展演算子

と呼びます。

---

# 時間発展演算子の意味

時間発展演算子は：

初期状態を未来へ進める操作

です。

つまり：

```text
|ψ(t)⟩ = U(t)|ψ(0)⟩
```

となります。

---

# なぜユニタリ演算子になるのか？

重要な性質：

時間発展演算子は

ユニタリ

です。

つまり：

```text
U†U = I
```

が成立します。

---

# なぜユニタリである必要があるのか？

理由：

確率保存

のためです。

量子状態のノルム：

```text
⟨ψ|ψ⟩
```

は：

時間とともに変化してはいけません。

そのため：

時間発展はユニタリになります。

---

# なぜ exp(-iHt) はユニタリになるのか？

理由：

Hamiltonianは

エルミート演算子

だからです。

つまり：

```text
H† = H
```

です。

このとき：

```text
exp(-iHt)
```

は必ずユニタリになります。

---

# 物理的な意味

時間発展とは：

状態ベクトルの回転

です。

例えば：

```text
H = Z
```

なら：

Bloch球上の回転になります。

---

# 単一量子ビットの例

例：

```text
H = Z
```

なら：

```text
U(t) = exp(-iZt)
```

になります。

これは：

RZ回転

と同じです。

---

# 回路としての表現

つまり：

```text
exp(-iZt)
```

は：

```text
RZ(2t)
```

として実装できます。

これが：

Hamiltonian simulationの基本です。

---

# もう少し一般的な例

例：

```text
H = X
```

なら：

```text
U(t) = exp(-iXt)
```

になります。

これは：

```text
RX(2t)
```

です。

---

# 相互作用Hamiltonianの場合

例：

```text
H = Z₀Z₁
```

なら：

```text
U(t) = exp(-iZ₀Z₁t)
```

になります。

これは：

CNOT + RZ + CNOT

で実装できます。

---

# なぜこれが重要なのか？

理由：

Hamiltonianが分かれば

回路が作れる

からです。

つまり：

物理モデル → 量子回路

へ変換できます。

---

# 複数項のHamiltonianの場合

例えば：

```text
H = X + Z
```

なら：

```text
U(t) = exp(-i(X+Z)t)
```

になります。

しかし：

ここで問題が発生します。

---

# 重要な問題

一般に：

```text
exp(A+B) ≠ exp(A)exp(B)
```

です。

つまり：

単純には分解できません。

---

# なぜ問題になるのか？

Hamiltonianは通常：

複数のPauli項の和

です。

つまり：

```text
H = Σ cᵢPᵢ
```

です。

そのため：

直接回路化できません。

---

# 解決方法

この問題を解決する方法が：

**Trotter分解**

です。

これは：

指数関数を近似的に分解する方法

です。

---

# Hamiltonian simulationの核心

量子シミュレーションとは：

次を実装することです：

```text
exp(-iHt)
```

そして：

これを近似的に実現するのが

Trotter分解

です。

---
