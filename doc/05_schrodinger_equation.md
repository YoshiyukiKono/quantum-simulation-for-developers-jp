# 第05回 シュレディンガー方程式と量子回路

## はじめに

前回は：

* シュレディンガー方程式
* 時間発展演算子
* 指数関数としての時間発展
* ユニタリ演算子の意味
* 単一Pauli項の回路表現
* 複数項Hamiltonianの課題

を整理しました。

今回は：

時間発展が

**どのように量子回路として実装されるのか**

を理解します。

つまり：

シュレディンガー方程式

と

量子回路

の関係を結びつけます。

---

# 量子回路は時間発展そのものである

重要な視点：

量子回路とは

時間発展の離散表現

です。

つまり：

量子回路 = 物理の時間変化

です。

---

# 連続時間と離散時間

物理では：

状態は連続的に変化します：

```text
|ψ(t)⟩
```

しかし：

量子回路では：

離散ステップで変化します：

```text
|ψ₀⟩ → |ψ₁⟩ → |ψ₂⟩ → ...
```

この違いが：

量子シミュレーションの出発点です。

---

# 回路とはユニタリ変換の列である

量子回路は：

ユニタリ演算の積

として表されます：

```text
U = Uₙ Uₙ₋₁ ... U₂ U₁
```

そして：

各ゲートは

小さな時間発展

に対応しています。

---

# 小さな時間発展としてのゲート

時間発展演算子：

```text
U(t) = exp(-iHt)
```

を：

小さな時間幅に分けると：

```text
U(Δt) = exp(-iHΔt)
```

になります。

この：

Δt の積み重ね

が

量子回路です。

---

# 離散化された時間発展

時間を分割すると：

```text
U(t) = (exp(-iHΔt))^n
```

になります。

ここで：

```text
t = nΔt
```

です。

つまり：

回路の深さ = 時間

になります。

---

# 回路深さと物理時間の関係

重要な対応関係：

回路深さが増える

↓

長時間の物理シミュレーション

が可能になる

という意味になります。

---

# 例：Z方向の時間発展

Hamiltonian：

```text
H = Z
```

の場合：

時間発展：

```text
U(t) = exp(-iZt)
```

これは：

回路では：

```text
RZ(2t)
```

になります。

つまり：

1ゲート = 時間発展

です。

---

# X方向の時間発展

同様に：

```text
H = X
```

なら：

```text
U(t) = exp(-iXt)
```

これは：

```text
RX(2t)
```

になります。

---

# 回転ゲートの物理的意味

つまり：

RX
RY
RZ

はすべて：

時間発展演算子

です。

これは：

非常に重要な理解です。

---

# 複数量子ビットの場合

例：

```text
H = Z₀Z₁
```

なら：

```text
U(t) = exp(-iZ₀Z₁t)
```

になります。

これは：

次の回路で実装できます：

```text
CNOT
RZ
CNOT
```

つまり：

相互作用も時間発展として実装可能です。

---

# なぜ相互作用が回路になるのか？

理由：

量子ゲートは

相互作用を再現する操作

だからです。

例えば：

CNOTは

2量子ビット相互作用

を表現しています。

---

# Hamiltonian simulationの本質

ここまでをまとめると：

量子シミュレーションとは：

次を実装することです：

```text
exp(-iHt)
```

そして：

量子回路とは：

その近似表現です。

---

# 任意のHamiltonianを回路にする方法

一般形：

```text
H = Σ cᵢPᵢ
```

の場合：

時間発展：

```text
exp(-iHt)
```

は：

```text
exp(-iΣ cᵢPᵢ t)
```

になります。

しかし：

ここで問題があります。

---

# 非可換性の問題

一般に：

```text
[Pᵢ , Pⱼ] ≠ 0
```

つまり：

順番を入れ替えられません。

そのため：

```text
exp(A+B) ≠ exp(A)exp(B)
```

になります。

これが：

量子シミュレーションの難しさです。

---

# それでも回路化したい

私たちの目的：

```text
exp(-iHt)
```

を回路に変換すること

です。

そのためには：

近似が必要になります。

---

# 近似の考え方

時間を小さく分けると：

近似が成立します：

```text
exp((A+B)Δt) ≈ exp(AΔt)exp(BΔt)
```

Δt が小さいほど：

精度が上がります。

---

# 小さな時間発展の積み重ね

つまり：

```text
exp((A+B)t)
```

は：

```text
(exp(AΔt)exp(BΔt))^n
```

として近似できます。

これが：

量子回路設計の基本思想

です。

---

# デジタル量子シミュレーション

この方法は：

デジタル量子シミュレーション

と呼ばれます。

特徴：

時間を分割する
ゲート列として実装する
誤差を制御できる

という利点があります。

---

# アナログ量子シミュレーションとの違い

対照的な方法：

アナログ量子シミュレーション

です。

これは：

物理系そのものを再現します。

一方：

量子回路方式は：

デジタルシミュレーション

です。

---

# なぜデジタル方式が重要なのか？

理由：

汎用量子コンピュータで実装できる

からです。

つまり：

Qiskitで実装できるのは

こちらです。

---

# Qiskitとの関係

Qiskitでは：

時間発展演算子は：

PauliEvolutionGate

などで実装できます。

つまり：

Hamiltonianを入力すると

回路に変換できます。

---

# 次に必要な技術

ここまでで：

次の問題が明確になりました：

```text
exp(A+B)
```

をどう扱うか？

その解決策が：

**Trotter分解**

です。
