# なぜ測定が重要なのか？

量子計算では：

状態そのものは直接読めません。

観測できるのは：

測定結果の統計

だけです。

つまり：

```text
1回の測定 → 不十分
多数回の測定 → 必要
```

になります。

---

# ショット（shot）とは何か？

ショットとは：

回路を実行する回数

です。

例：

```text
shots = 1000
```

なら：

1000回測定します。

---

# なぜショットが必要なのか？

理由：

量子測定は確率的

だからです。

例：

```text
|ψ⟩ = 0.8|0⟩ + 0.6|1⟩
```

なら：

1回では：

0 または 1

しか観測できません。

---

# 期待値とは何か？

VQEやQAOAでは：

期待値を計算します：

```text
⟨H⟩
```

これは：

平均値

です。

つまり：

```text
多数回測定
↓
平均
↓
期待値
```

になります。

---

# 測定誤差の大きさ

統計誤差は：

次で決まります：

```text
誤差 ≈ 1 / √shots
```

つまり：

| shots | 誤差 |
| ----- | -- |
| 100   | 大  |
| 1000  | 中  |
| 10000 | 小  |

です。

---

# なぜshot数を増やしすぎないのか？

理由：

実機計算は高価

だからです。

つまり：

```text
shots ↑ → 精度 ↑
shots ↑ → コスト ↑
```

になります。

---

# Hamiltonian期待値の測定

Hamiltonianは通常：

複数のPauli項の和です：

```text
H = c₁P₁ + c₂P₂ + c₃P₃ + ...
```

期待値：

```text
⟨H⟩ = c₁⟨P₁⟩ + c₂⟨P₂⟩ + ...
```

となります。

---

# 問題：各Pauli項を個別に測定する必要がある

例：

```text
Z₀Z₁
X₀X₁
Y₀Y₁
```

は：

同時測定できません。

つまり：

```text
測定回数が増える
```

という問題があります。

---

# 解決方法：Pauli grouping

Pauli groupingとは：

同時測定可能な演算子をまとめる

方法です。

例：

```text
Z₀Z₁
Z₁Z₂
Z₂Z₃
```

は：

同時測定できます。

---

# なぜ同時測定できるのか？

理由：

可換だからです：

```text
[P₁, P₂] = 0
```

なら：

同時測定可能

です。

---

# 可換性の重要性

重要なルール：

```text
可換 → 同時測定可能
非可換 → 個別測定必要
```

です。

---

# Pauli groupingの効果

例：

| 方法         | 必要測定回数 |
| ---------- | ------ |
| groupingなし | 30回    |
| groupingあり | 5回     |

になります。

つまり：

測定コスト削減

が可能です。

---

# Tensor Product Basis grouping

代表的手法：

Tensor Product Basis（TPB）

です。

例：

```text
ZIZ
ZZZ
IZZ
```

は：

同時測定できます。

---

# グループ化の直感

次のように分類します：

```text
Z測定グループ
X測定グループ
Y測定グループ
```

これだけでも：

大幅削減できます。

---

# 分散（variance）とは何か？

期待値の誤差は：

分散で決まります：

```text
Var(H)
```

つまり：

```text
分散が大きい → 多くのshots必要
分散が小さい → 少ないshotsで十分
```

です。

---

# 分散最適化とは何か？

各Pauli項に：

異なるshot数を割り当てる方法です。

つまり：

```text
重要な項 → 多く測定
重要でない項 → 少なく測定
```

になります。

---

# Weighted shot allocation

例：

```text
|係数| が大きい項 → 多く測定
|係数| が小さい項 → 少なく測定
```

です。

これは：

効率的な測定戦略

になります。

---

# VQEにおける測定最適化

VQEでは：

期待値評価が支配的コスト

です。

つまり：

```text
回路実行時間
<
測定時間
```

になることが多いです。

---

# QAOAにおける測定最適化

QAOAでは：

ビット列分布

が重要です。

つまり：

```text
最頻値探索
```

が目的になります。

そのため：

shot数設計が重要です。

---

# Adaptive measurementとは何か？

Adaptive measurement：

測定途中で

shot配分を変更する方法

です。

つまり：

```text
途中結果
↓
重要項を再測定
```

となります。

---

# Classical shadowとは何か？

Classical shadowとは：

少ない測定回数で

多数の期待値を推定する方法

です。

特徴：

```text
測定回数削減
```

が可能です。

---

# なぜ重要なのか？

理由：

NISQでは：

測定が最大コスト

だからです。

つまり：

```text
回路設計より測定設計が重要
```

になる場合もあります。

---

# Qiskitでshot数を指定する

例：

```python
sampler.run(circuit, shots=1024)
```

これで：

shot数を制御できます。

---

# 実機向けshot設計の目安

典型例：

| 用途    | shots |
| ----- | ----- |
| デバッグ  | 100   |
| VQE試験 | 1000  |
| 実機評価  | 10000 |

です。

---

# 測定戦略の設計手順

基本手順：

```text
Hamiltonian分解
↓
可換グループ化
↓
shot配分最適化
↓
Error mitigation適用
```

です。

---

# NISQ測定設計の基本原則

重要：

```text
少ない測定で
多くの情報を得る
```

です。

これは：

量子アルゴリズム設計の核心

の一つです。
